# ADR-003: Spring Events (Observer) para notificaciones desacopladas

## Estado
Aceptado — 2026-05-07

## Contexto
El servicio de pedidos necesitaba notificar a múltiples receptores (email, log, futuro SMS) cuando un pedido era procesado. El acoplamiento directo a `JavaMailSender` en el servicio principal mezclaba responsabilidades y hacía frágil el servicio central ante cambios en los canales de notificación.

## Decisión
Se usa `ApplicationEventPublisher` de Spring (implementación del patrón Observer) para publicar `PedidoProcesadoEvent`. Cada canal de notificación implementa un `@EventListener` independiente (`NotificacionEmail`, `NotificacionLog`). La `FachadaPedidos` solo publica el evento sin conocer quién lo escucha.

## Consecuencias

**Positivas:**
- Agregar un nuevo canal de notificación no requiere modificar `FachadaPedidos`.
- Los listeners son opcionales: si no hay suscriptores, el evento se ignora sin error.
- Cada listener es testeable de forma independiente.

**Negativas:**
- El orden de ejecución de los listeners no está garantizado.
- El debugging es más complejo: hay que rastrear todos los listeners para entender qué ocurre tras publicar el evento.
