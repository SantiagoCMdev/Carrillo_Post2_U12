# ADR-001: Arquitectura Hexagonal para aislar el dominio

## Estado
Aceptado — 2026-05-07

## Contexto
El sistema de pedidos debe soportar múltiples tipos de procesamiento y canales de notificación. El acoplamiento directo del servicio a Spring Data JPA y JavaMailSender dificultaba las pruebas unitarias y hacía costoso cambiar la implementación de infraestructura. Se necesitaba una arquitectura que permitiera probar el dominio sin levantar el contenedor de Spring.

## Decisión
Se adopta Arquitectura Hexagonal. El dominio define puertos (interfaces) en el paquete `dominio.puertos`. Los adaptadores e infraestructura implementan esos puertos. El dominio no importa ninguna clase de Spring ni de JPA directamente.

## Consecuencias

**Positivas:**
- El dominio es testeable con Mockito sin necesitar el contenedor Spring.
- Cambiar de JPA a MongoDB requiere solo crear un nuevo adaptador sin tocar el dominio.
- Las reglas ArchUnit verifican automáticamente que el aislamiento se mantiene.

**Negativas:**
- Más interfaces y clases de mapeo que en MVC clásico.
- Curva de aprendizaje para desarrolladores acostumbrados a arquitecturas en capas simples.
